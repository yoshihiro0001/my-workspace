# 開発キックオフ（オーナーの一言 + エージェントへの第一声）

`docs/開発依頼書.md` をリポジトリに置いたうえで、**実装を始めるとき**に使います。  
テンプレの **`docs/FIRST_AGENT_PROMPT.md`（名前・デプロイの一括セットアップ）をまだなら先に済ませてください。**

---

## 1. オーナーが最初に書く（自分も添える）

AI に送る **直前** に、次を **自分の言葉で埋めてください**（1行でも可）。

- **いま一番達成したいこと（1〜3文）:**  
  _（例: まずトップにプロジェクトカード一覧だけ。ダミーデータでよい）_

- **絶対に外したくないこと:**  
  _（例: 保存・戻すの安心感／スマホで迷わない）_

- **後回しでよいこと:**  
  _（例: 本番の AI API・DB は Phase 3 以降）_

---

## 2. エージェントに送る「開発開始」コピペ用

セクション 1 を書いたら、下の枠をコピーし、**【オーナーの意図】** の行を自分の文章に差し替えて送ってください。

```
このリポジトリで docs/開発依頼書.md の仕様どおり「AI搭載開発ワークスペース」を実装していく。

【必読・この順】
00_START_HERE.md → AGENTS.md → docs/開発依頼書.md → docs/00_constitution.md → docs/01_history.md → docs/02_operations.md → docs/DESIGN_HANDOVER.md

【オーナーの意図】
（※ここに DEVELOPMENT_KICKOFF.md のセクション1を貼る）

【やってほしいこと】
- docs/00_constitution.md を開発依頼書と矛盾しないよう必要なら更新する。
- Phase 1 のゴールを1文で提案し、合意のうえ実装する（動く最小単位から）。
- 終了時に docs/01_history.md に追記する。

憲法・履歴・デザイン憲法に従って実装し、終わったら docs/01_history.md に追記して。
```

---

## 補足

- 日常の細かい依頼では、**`docs/ONE_LINER.md` の一言**を依頼の末尾に付ければ足ります。
- 仕様の正本は **`docs/開発依頼書.md`** です（テンプレの `DEVELOPMENT_BRIEF.md` と同内容に揃えてもよい）。
